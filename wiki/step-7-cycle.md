# The Step-7 Cycle

Start on any step-number. Add 7. Take mod 12. Repeat. You will visit **all 12 chromatic steps** before returning to where you started. This is the step-7 cycle ‚Äî the organizing structure that emerges from repeatedly stacking the interval with the simplest frequency ratio after the octave (‚âà 3:2, or the 7-step-interval in [12-TET](twelve-tet.md)).

Western theory calls this the "circle of fifths." PhizMusic calls it what it is: a cycle generated by the 7-step-interval under modular arithmetic.

> üéØ **Simple version**: Pick any note. Jump 7 keys up on a piano (counting every key, black and white). Keep jumping by 7. You'll land on every single note exactly once before coming back to where you started. That's the step-7 cycle ‚Äî it's the only "tour" of all 12 notes where each jump is the most harmonically powerful interval available.

## The Mathematics: Modular Arithmetic

The step-7 cycle is a direct consequence of arithmetic in the integers modulo 12 (written ‚Ñ§‚ÇÅ‚ÇÇ). Here is the procedure:

1. Start at step-number 0.
2. Add 7.
3. If the result is ‚â• 12, subtract 12 (i.e., take mod 12).
4. Repeat.

### The full sequence

| Iteration (n) | Calculation | (n √ó 7) mod 12 | Dodeka syllable |
|:-:|:-:|:-:|:-:|
| 0 | 0 √ó 7 = 0 | **0** | Do |
| 1 | 1 √ó 7 = 7 | **7** | So |
| 2 | 2 √ó 7 = 14 ‚Üí 14 ‚àí 12 | **2** | Re |
| 3 | 3 √ó 7 = 21 ‚Üí 21 ‚àí 12 | **9** | La |
| 4 | 4 √ó 7 = 28 ‚Üí 28 ‚àí 24 | **4** | Mi |
| 5 | 5 √ó 7 = 35 ‚Üí 35 ‚àí 24 | **11** | Si |
| 6 | 6 √ó 7 = 42 ‚Üí 42 ‚àí 36 | **6** | Hu |
| 7 | 7 √ó 7 = 49 ‚Üí 49 ‚àí 48 | **1** | Ka |
| 8 | 8 √ó 7 = 56 ‚Üí 56 ‚àí 48 | **8** | Bi |
| 9 | 9 √ó 7 = 63 ‚Üí 63 ‚àí 60 | **3** | Xo |
| 10 | 10 √ó 7 = 70 ‚Üí 70 ‚àí 60 | **10** | Ve |
| 11 | 11 √ó 7 = 77 ‚Üí 77 ‚àí 72 | **5** | Fa |
| 12 | 12 √ó 7 = 84 ‚Üí 84 ‚àí 84 | **0** | Do ‚Üê back to start |

The cycle visits every step-number exactly once: **0 ‚Üí 7 ‚Üí 2 ‚Üí 9 ‚Üí 4 ‚Üí 11 ‚Üí 6 ‚Üí 1 ‚Üí 8 ‚Üí 3 ‚Üí 10 ‚Üí 5 ‚Üí (0)**.

### Why does 7 generate all 12?

The key property: **gcd(7, 12) = 1** ‚Äî that is, 7 and 12 share no common factors. They are *coprime*.

In the group ‚Ñ§‚ÇÅ‚ÇÇ (integers mod 12 under addition), an element *g* generates the entire group if and only if gcd(*g*, 12) = 1. The proof is straightforward: if gcd(*g*, 12) = *d* > 1, then all multiples of *g* mod 12 are also multiples of *d*, so you can only reach 12/*d* distinct values ‚Äî not all 12.

The **generators** of ‚Ñ§‚ÇÅ‚ÇÇ are exactly the elements coprime to 12:

```
Generators:  1, 5, 7, 11   (each visits all 12 steps)
```

Step-intervals 1 and 11 are trivial (chromatic steps up/down). Step-intervals **5 and 7** are the musically interesting generators ‚Äî and they are related: 12 ‚àí 7 = 5. Moving by the 7-step-interval clockwise is the same as moving by the 5-step-interval counterclockwise.

### Step-intervals that do NOT generate all 12

What happens when gcd(*g*, 12) > 1? The cycle closes early:

| Step-interval (g) | gcd(g, 12) | Steps generated | Cycle |
|:-:|:-:|:-:|:--|
| 2 | 2 | 6 of 12 | {0, 2, 4, 6, 8, 10} ‚Äî whole-tone subset |
| 3 | 3 | 4 of 12 | {0, 3, 6, 9} ‚Äî diminished subset |
| 4 | 4 | 3 of 12 | {0, 4, 8} ‚Äî augmented subset |
| 6 | 6 | 2 of 12 | {0, 6} ‚Äî tritone pair |

Each generates only 12 / gcd(*g*, 12) distinct steps. These partial cycles are themselves musically significant ‚Äî they correspond to the equal divisions of the octave ‚Äî but they cannot serve as generators of the full chromatic set.

> üéØ **Simple version**: 7 and 12 don't share any divisors (besides 1), so jumping by 7 never falls into a repeating loop until you've hit every note. Jump by 2 instead, and you only hit the even-numbered notes ‚Äî you're stuck in a 6-note loop forever.

### The formula, summarized

For any step-interval *g* and number of iterations *n*:

```
step-number = (n √ó g) mod 12
```

If gcd(*g*, 12) = 1, this visits all 12 step-numbers for n = 0, 1, ‚Ä¶, 11.

## Why the 7 Step-Interval?

Among the four generators of ‚Ñ§‚ÇÅ‚ÇÇ (1, 5, 7, 11), the 7-step-interval holds a privileged acoustic position. Its frequency ratio in [12-TET](twelve-tet.md) is:

```
2^(7/12) ‚âà 1.4983
```

This is the closest 12-TET interval to the **3:2 ratio** (= 1.5000) ‚Äî the simplest frequency ratio after the octave (2:1). The 3:2 ratio is acoustically special because:

1. **Harmonic fusion.** A tone's 3rd harmonic (3*f*) and another tone's 2nd harmonic (2 √ó 1.5*f* = 3*f*) coincide exactly. This spectral overlap creates the sensation of *consonance* ‚Äî the two tones fuse perceptually. See [Consonance & Dissonance](consonance-dissonance.md).

2. **Simplicity of ratio.** After 1:1 (unison) and 2:1 (octave), the ratio 3:2 uses the smallest possible integers. Simpler ratios ‚Üí more harmonic overlap ‚Üí stronger perceptual fusion.

3. **Universality.** The 3:2 ratio appears in virtually every tuning system across cultures ‚Äî Pythagorean tuning, Chinese *sƒÅnfƒìn s«îny√¨* (‰∏âÂàÜÊêçÁõä), Indian *·π£a·∏çja-pa√±cama* relationship ‚Äî because it is a mathematical fact about vibrating objects, not a cultural preference.

The 5-step-interval (the generator in the opposite direction) approximates the **4:3 ratio** ‚Äî the octave complement of 3:2 (since 2/1 √∑ 3/2 = 4/3). It is equally valid as a generator, and produces the same cycle in reverse order.

## The Cycle as a Map

The step-7 cycle is not just a mathematical curiosity ‚Äî it is a **map** that reveals deep structure in how pitch-subsets relate to each other.

### Adjacent steps = the 7-step-interval apart

On the cycle, each position is exactly 7 chromatic steps from its neighbors. This means proximity on the cycle indicates *harmonic closeness* ‚Äî pitches that share strong spectral overlap.

### Contiguous arcs = common step-subsets

Select any **7 consecutive positions** on the cycle and collect their step-numbers. You get a set of 7 steps from the 12 available ‚Äî and this set is always one of the [diatonic step-subsets](scales.md) (what Western theory calls major/minor scales, modes):

```
7 consecutive from position 0: {0, 7, 2, 9, 4, 11, 6} ‚Üí sorted: {0, 2, 4, 6, 7, 9, 11}
7 consecutive from position 1: {7, 2, 9, 4, 11, 6, 1} ‚Üí sorted: {1, 2, 4, 6, 7, 9, 11}
... and so on for all 12 starting positions
```

Any **5 consecutive positions** yield a pentatonic step-subset. Any **6 consecutive** yield a hexatonic. The cycle organizes step-subsets by how many positions they share ‚Äî the more overlap on the cycle, the more closely related the subsets sound.

### Clockwise vs. counterclockwise

- **Clockwise** = adding the 7-step-interval (ascending by ‚âà 3:2)
- **Counterclockwise** = adding the 5-step-interval (ascending by ‚âà 4:3)

Both traversals visit all 12 steps. They are mirror images of the same mathematical structure.

window.addEventListener('load', function() {
(function () {
  var DODEKA = ["Do", "Ka", "Re", "Xo", "Mi", "Fa", "Hu", "So", "Bi", "La", "Ve", "Si"];
  var BASE_FREQ = 220;

  var canvas = document.getElementById("s7-cycle-canvas");
  var stepBtn = document.getElementById("s7-step-btn");
  var resetBtn = document.getElementById("s7-reset-btn");
  var playToggle = document.getElementById("s7-play-toggle");
  var infoEl = document.getElementById("s7-info");

  var generatorBtns = [];
  var generatorValues = [7, 5, 2, 3, 4, 6];
  var generatorIds = ["s7-gen-7", "s7-gen-5", "s7-gen-2", "s7-gen-3", "s7-gen-4", "s7-gen-6"];
  for (var b = 0; b < generatorIds.length; b++) {
    generatorBtns.push(document.getElementById(generatorIds[b]));
  }

  var generator = 7;
  var currentN = 0;
  var visitedSteps = [];
  var stopped = false;
  var maxSteps = 12;

  function gcd(a, b) {
    a = Math.abs(a);
    b = Math.abs(b);
    while (b) {
      var t = b;
      b = a % b;
      a = t;
    }
    return a;
  }

  function computeMaxSteps(g) {
    return 12 / gcd(g, 12);
  }

  function computeStep(n, g) {
    return (n * g) % 12;
  }

  function setGenerator(g) {
    generator = g;
    maxSteps = computeMaxSteps(g);
    for (var i = 0; i < generatorBtns.length; i++) {
      if (generatorValues[i] === g) {
        generatorBtns[i].className = "active";
      } else {
        generatorBtns[i].className = "";
      }
    }
    doReset();
  }

  function doReset() {
    currentN = 0;
    visitedSteps = [];
    stopped = false;
    updateInfo();
    redraw();
  }

  function doStep() {
    if (stopped) return;
    if (currentN >= maxSteps) return;

    var step = computeStep(currentN, generator);
    visitedSteps.push(step);

    if (playToggle.checked) {
      playTone(step);
    }

    currentN++;

    if (currentN >= maxSteps) {
      if (maxSteps < 12) {
        stopped = true;
      }
    }

    updateInfo();
    redraw();
  }

  function updateInfo() {
    if (visitedSteps.length === 0) {
      infoEl.textContent = "Press \"Step through\" to begin. Generator: step-interval " + generator;
      return;
    }

    var lastIdx = visitedSteps.length - 1;
    var n = lastIdx;
    var step = visitedSteps[lastIdx];
    var line1 = "Step " + (n + 1) + ": (" + n + " \u00D7 " + generator + ") mod 12 = " + step + " | Dodeka: " + DODEKA[step];

    var pathParts = [];
    for (var i = 0; i < visitedSteps.length; i++) {
      pathParts.push(String(visitedSteps[i]));
    }
    var line2 = "Path: " + pathParts.join(" \u2192 ");

    if (stopped && maxSteps < 12) {
      var g = gcd(generator, 12);
      var line3 = "\u26A0 Step-interval " + generator + " generates only " + maxSteps + " of 12 steps (gcd(" + generator + ", 12) = " + g + ")";
      infoEl.textContent = line1 + "\n" + line2 + "\n" + line3;
    } else if (currentN >= maxSteps && maxSteps === 12) {
      infoEl.textContent = line1 + "\n" + line2 + "\n\u2705 All 12 steps visited! Cycle complete.";
    } else {
      infoEl.textContent = line1 + "\n" + line2;
    }
  }

  function redraw() {
    if (typeof PhizViz !== "undefined" && PhizViz.drawStep7Cycle) {
      PhizViz.drawStep7Cycle(canvas, {
        highlight: visitedSteps,
        showArrows: true
      });
    }
  }

  function playTone(stepNumber) {
    if (typeof Tone === "undefined") return;
    Tone.start().then(function () {
      var freq = BASE_FREQ * Math.pow(2, stepNumber / 12);
      var gain = new Tone.Gain(0.3).toDestination();
      var osc = new Tone.Oscillator(freq, "sine").connect(gain);
      osc.start();
      setTimeout(function () {
        osc.stop();
        osc.dispose();
        gain.dispose();
      }, 500);
    });
  }

  // --- Event listeners ---

  stepBtn.addEventListener("click", function () {
    doStep();
  });

  resetBtn.addEventListener("click", function () {
    doReset();
  });

  for (var g = 0; g < generatorBtns.length; g++) {
    generatorBtns[g].addEventListener("click", (function (val) {
      return function () { setGenerator(val); };
    })(generatorValues[g]));
  }

  // --- Initial state ---
  setGenerator(7);

  redraw();
})();
});
